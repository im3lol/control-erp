# نظام تخطيط موارد المؤسسات (ERP)
## دليل المستخدم والتقسيم الوظيفي

---

## ١. نظرة شاملة على النظام

النظام عبارة عن منصة متكاملة لإدارة العمليات اليومية للشركة، بدءاً من المخزون وصولاً إلى التقارير المالية. يتكون من ٦ وحدات رئيسية تعمل معاً كمنظومة واحدة.

```
┌─────────────────────────────────────────────────────────────┐
│                    النظام المتكامل                           │
├─────────────┬─────────────┬─────────────┬───────────────────┤
│  الإعدادات  │   المخازن   │   الحسابات  │     المبيعات      │
│  SETTINGS   │  INVENTORY  │  ACCOUNTING │      SALES        │
├─────────────┼─────────────┼─────────────┼───────────────────┤
│  المشتريات  │   التقارير  │             │                   │
│  PURCHASE   │   REPORTS   │             │                   │
└─────────────┴─────────────┴─────────────┴───────────────────┘
```

**الفكرة الأساسية:** أي حركة في النظام (بيع، شراء، تسوية) تؤثر تلقائياً على ثلاثة أماكن:
- المخزن (الكميات)
- الحسابات (القيود المحاسبية)
- العملاء/الموردين (الأرصدة)

---

## ٢. وحدة الإعدادات

### الغرض
هي الأساس الذي يقوم عليه النظام. بدونها لا يمكن تشغيل أي وحدة أخرى.

### العناصر

| العنصر | الوظيفة | مثال |
|--------|---------|------|
| بيانات الشركة | اسم الشركة، العنوان، الرقم الضريبي، الشعار | شركة الأمل للتجارة |
| العملات | تحديد العملات المستخدمة وأسعار الصرف | جنيه مصري (أساسي)، دولار |
| وحدات القياس | كيفية قياس الأصناف | قطعة، كيلو، لتر، صندوق |
| المستخدمين | من يستخدم النظام وصلاحياته | مدير، محاسب، بائع، مخزن |

### الصلاحيات

```
المدير (Admin)
├── يدخل على كل حاجة
├── يضيف/يعدل/يحذف مستخدمين
└── يغير إعدادات الشركة

المحاسب (Accountant)
├── شجرة الحسابات
├── القيود اليومية
└── التقارير المالية

البائع (Sales)
├── العملاء
├── فواتير البيع
└── التحصيل

مستلم المشتريات (Purchase)
├── الموردين
├── فواتير الشراء
└── السداد

أمين المخزن (Inventory)
├── الأصناف
├── المخازن
└── حركات المخزن

المشاهد فقط (Viewer)
└── يشوف بس (تقارير، أرصدة)
```

## ٣. وحدة المخازن

### الغرض
معرفة "إيه اللي عندك؟ وفين؟ وبكام؟"

### العناصر

#### ٣.١ المخزن
المكان الفعلي اللي بتحط فيه البضاعة.

| المخزن | الموقع | المسؤول |
|--------|--------|---------|
| مخزن رئيسي | القاهرة | أحمد |
| مخزن فرعي | الإسكندرية | محمد |

#### ٣.٢ الصنف
أي حاجة بتتعامل معاها: منتج، خامة، قطعة غيار.

| الصنف | الكود | الفئة | الوحدة | طريقة التكلفة |
|-------|-------|-------|--------|---------------|
| تيشيرت أبيض | TSH-001 | منتجات | قطعة | FIFO |
| قماش قطن | FAB-001 | خامات | متر | FIFO |

**طريقة التكلفة (FIFO):** "اللي دخل أول، يطلع أول"
- لو اشتريت ١٠٠ قطعة بـ ١٠ جنيه في يناير
- واشتريت ٥٠ قطعة بـ ١٢ جنيه في فبراير
- وبيعت ١٢٠ قطعة في مارس
- التكلفة = (١٠٠ × ١٠) + (٢٠ × ١٢) = ١٠٠٠ + ٢٤٠ = ١٢٤٠ جنيه

#### ٣.٣ حركة المخزن
أي تغيير في الكمية.

| النوع | المعنى | يحصل إمتى |
|-------|--------|-----------|
| دخول (IN) | زيادة في المخزن | شراء، رصيد افتتاحي، تسوية (+) |
| خروج (OUT) | نقصان في المخزن | بيع، تسوية (-)، هالك |
| تسوية (ADJ) | تصحيح الرصيد | جرد فعلي مختلف عن النظام |

#### ٣.٤ رصيد الصنف
كمية كل صنف في كل مخزن.

```
┌─────────────┬─────────────────┬──────────┬────────────┐
│   الصنف     │     المخزن      │ الكمية   │ متوسط التكلفة │
├─────────────┼─────────────────┼──────────┼────────────┤
│ تيشيرت أبيض │ مخزن رئيسي      │   ٥٠٠    │    ١١.٥    │
│ تيشيرت أبيض │ مخزن الإسكندرية │   ٢٠٠    │    ١١.٥    │
│ قماش قطن    │ مخزن رئيسي      │  ١٠٠٠    │     ٨      │
└─────────────┴─────────────────┴──────────┴────────────┘
```

### القواعد
- مش مسموح بالرصيد السالب (ماينفعش تبيع اللي مش عندك)
- أي حركة لازم يكون ليها سبب (فاتورة، تسوية، إلخ)
- التكلفة بتتحسب تلقائياً حسب طريقة FIFO

## ٤. وحدة الحسابات

### الغرض
تسجيل كل حركة مالية بطريقة القيد المزدوج: "لكل مدين مقابل"

### العناصر

#### ٤.١ شجرة الحسابات
تصنيف كل الحسابات في هيكل شجري.

```
١- الأصول (اللي بتملكه الشركة)
├── ١١- أصول متداولة (تتحول لنقد في سنة)
│   ├── ١١٠١- النقدية (الكاش)
│   ├── ١١٠٢- البنوك
│   ├── ١١٠٣- العملاء (اللي عليهم فلوس)
│   └── ١١٠٤- المخزون
└── ١٢- أصول ثابتة (معدات، عربيات)

٢- الخصوم (اللي عليك)
├── ٢١- خصوم متداولة (تسددها في سنة)
│   ├── ٢١٠١- الموردين (اللي ليهم فلوس)
│   └── ٢١٠٢- الضريبة المستحقة

٣- حقوق الملكية
├── ٣١- رأس المال
└── ٣٢- الأرباح المحتجزة

٤- الإيرادات (الفلوس اللي داخلة)
└── ٤١- مبيعات

٥- المصروفات (الفلوس اللي طالعة)
├── ٥١- تكلفة البضاعة المباعة
└── ٥٢- مصروفات تشغيل (إيجار، مرتبات)
```

#### ٤.٢ القيد اليومي
المستند المحاسبي الأساسي. كل قيد فيه سطرين على الأقل: مدين وداين.

**مثال: شراء بضاعة بـ ١٠٠٠ جنيه من مورد**

| الحساب | مدين | داين | التفسير |
|--------|------|------|---------|
| المخزون (١١٠٤) | ١٠٠٠ | | زاد المخزن |
| الموردين (٢١٠١) | | ١٠٠٠ | علينا للمورد |

**مثال: بيع بضاعة بـ ١٥٠٠ جنيه لعميل (تكلفتها ١٠٠٠)**

| الحساب | مدين | داين | التفسير |
|--------|------|------|---------|
| العملاء (١١٠٣) | ١٥٠٠ | | العميل عليه |
| المبيعات (٤١) | | ١٥٠٠ | إيراد |
| تكلفة البضاعة (٥١) | ١٠٠٠ | | المصروف |
| المخزون (١١٠٤) | | ١٠٠٠ | نقص المخزن |

#### ٤.٣ حالات القيد

| الحالة | المعنى | إمتى يتغير |
|--------|--------|------------|
| مسودة | لسه مش نهائي | أول ما تعمله |
| مرحل | اتأكد واتسجل | لما تدوس "ترحيل" |
| معكوس | اتلغي بقيد عكسي | لما تحتاج تصحيح |

### القواعد
- القيد لازم يكون متوازن: مجموع المدين = مجموع الداين
- مش مسموح بالقيد على حساب رئيسي (لازم حساب فرعي)
- القيد المرحل مينفعش تعديله (لازم تعكسه وتعمل قيد جديد)

## ٥. وحدة المبيعات

### الغرض
إدارة علاقتك بالعملاء: من الفاتورة لحد التحصيل.

### العناصر

#### ٥.١ العميل

| البيان | الوصف |
|--------|-------|
| كود | رقم مميز للعميل (C-001) |
| الاسم | اسم العميل أو الشركة |
| التليفون | للتواصل |
| الرصيد | كام ليه أو عليه دلوقتي |
| حد الائتمان | أقصى مبلغ نبيعه له بالآجل |
| شروط الدفع | هيدفع بعد كام يوم (٣٠، ٦٠، ٩٠) |

#### ٥.٢ فاتورة البيع
رحلة الفاتورة من الإنشاء للتحصيل:

```
┌─────────┐    ┌──────────┐    ┌─────────────┐    ┌────────┐
│  مسودة  │───▶│  مؤكدة   │───▶│ مدفوع جزئي │───▶│ مدفوع  │
│ (DRAFT) │    │(CONFIRMED│    │(PARTIAL_PAID│    │ (PAID) │
└─────────┘    └──────────┘    └─────────────┘    └────────┘
      │               │
      ▼               ▼
┌─────────┐    ┌──────────┐
│  ملغية  │    │  ملغية   │
│(CANCELLED│   │(CANCELLED│
└─────────┘    └──────────┘
```

**لما تؤكد الفاتورة (Confirm):**
- بتخصم من المخزن
- بتعمل قيد محاسبي: العملاء ← المبيعات + الضريبة
- بتعمل قيد: تكلفة البضاعة ← المخزون
- بتحدث رصيد العميل

**لما العميل يدفع:**
- بتعمل قيد: النقدية ← العملاء
- بتحدث رصيد العميل
- بتقلل المبلغ المستحق على الفاتورة

#### ٥.٣ التحصيل

| نوع الدفع | إمتى تستخدمه |
|-----------|-------------|
| نقدي | لما العميل يدفع كاش |
| بنكي | تحويل بنكي |
| شيك | شيك بنكي |

**التوزيع:** المبلغ ممكن يتوزع على أكتر من فاتورة.

### القواعد
- مش مسموح تأكيد فاتورة لو الرصيد مش كافي
- لو العميل وصل لحد الائتمان، النظام يمنع البيع الآجل
- الفاتورة المؤكدة مينفعش تعديلها (لازم تلغيها وتعمل جديدة)

---

## ٦. وحدة المشتريات

### الغرض
إدارة علاقتك بالموردين: من أمر الشراء لحد السداد.

### العناصر

#### ٦.١ المورد
نفس فكرة العميل بس من الجهة التانية.

| البيان | الوصف |
|--------|-------|
| كود | رقم مميز (S-001) |
| الاسم | اسم المورد أو الشركة |
| الرصيد | كام لينا أو علينا |
| شروط الدفع | هنسدد بعد كام يوم |

#### ٦.٢ فاتورة الشراء

**لما تؤكد فاتورة الشراء:**
- بتضيف للمخزن (دخول)
- بتعمل طبقة تكلفة FIFO جديدة
- بتعمل قيد: المخزون + الضريبة ← الموردين
- بتحدث رصيد المورد

**لما تسدد للمورد:**
- بتعمل قيد: الموردين ← النقدية
- بتحدث رصيد المورد
- بتقلل المبلغ المستحق على الفاتورة

### الفرق بين البيع والشراء

| البيع | الشراء |
|-------|--------|
| بتخصم من المخزن | بتضيف للمخزن |
| العميل بيدينا فلوس | إحنا بندفع فلوس |
| قيد: العملاء ← مبيعات | قيد: مخزون ← موردين |
| COGS: تكلفة ← مخزون | FIFO layer جديدة |

## ٧. وحدة التقارير

### الغرض
تجميع البيانات في صورة مفهومة لاتخاذ القرار.

### أنواع التقارير

#### ٧.١ التقارير المالية

| التقرير | بيعمل إيه | مين يستخدمه |
|---------|----------|------------|
| ميزان المراجعة | كل حساب ومدينه وداينه | المحاسب |
| الميزانية العمومية | الأصول = الخصوم + حقوق الملكية | الإدارة |
| قائمة الدخل | ربحت ولا خسرت؟ | الإدارة |

#### ٧.٢ تقارير المخازن

| التقرير | بيعمل إيه |
|---------|----------|
| رصيد المخزن | كل صنف وكميته في كل مخزن |
| حركات المخزن | كل اللي دخل وطلق |
| تقييم المخزن | قيمة البضاعة بالتكلفة |
| تنبيه الطلب | الأصناف اللي وصلت لحد الأدنى |

#### ٧.٣ تقارير المبيعات والمشتريات

| التقرير | بيعمل إيه |
|---------|----------|
| ملخص المبيعات | فواتير البيع في فترة |
| تقرير العملاء | مين عليه ومن قد إيه |
| ملخص المشتريات | فواتير الشراء في فترة |
| تقرير الموردين | مين لينا ومن قد إيه |

### مثال: تقرير العملاء (Aging)

```
┌─────────────┬────────┬─────────┬─────────┬─────────┬─────────┐
│   العميل    │ الإجمالي│ الحالي  │ ٣٠ يوم │ ٦٠ يوم │ ٩٠+ يوم │
├─────────────┼────────┼─────────┼─────────┼─────────┼─────────┤
│ شركة النور  │ ٥٠,٠٠٠ │ ٢٠,٠٠٠  │ ١٥,٠٠٠ │ ١٠,٠٠٠  │ ٥,٠٠٠   │
│ محل الأمل   │ ٣٠,٠٠٠ │ ١٠,٠٠٠  │ ١٠,٠٠٠ │ ٥,٠٠٠   │ ٥,٠٠٠   │
└─────────────┴────────┴─────────┴─────────┴─────────┴─────────┘
```

## ٨. التدفقات الرئيسية (Workflows)

### ٨.١ دورة البيع الكاملة

```
١. إضافة عميل جديد (لو مش موجود)
   ↓
٢. إنشاء فاتورة بيع (مسودة)
   - اختيار العميل
   - إضافة أصناف (صنف + كمية + سعر)
   - النظام يحسب: المجموع + الخصم + الضريبة = الإجمالي
   ↓
٣. تأكيد الفاتورة
   - النظام يتأكد من توفر الرصيد
   - يخصم من المخزن
   - يعمل القيود المحاسبية
   - يحدث رصيد العميل
   ↓
٤. تحصيل من العميل
   - إنشاء سند قبض
   - توزيع المبلغ على الفواتير
   - يعمل قيد: نقدية ← عملاء
   ↓
٥. طباعة الفاتورة (A4)
```

### ٨.٢ دورة الشراء الكاملة

```
١. إضافة مورد جديد (لو مش موجود)
   ↓
٢. إنشاء فاتورة شراء (مسودة)
   - اختيار المورد
   - اختيار المخزن المستلم
   - إضافة أصناف
   ↓
٣. تأكيد الفاتورة
   - يضيف للمخزن
   - يعمل طبقة FIFO
   - يعمل قيد: مخزون ← موردين
   ↓
٤. سداد للمورد
   - إنشاء سند صرف
   - يعمل قيد: موردين ← نقدية
   ↓
٥. طباعة الفاتورة
```

### ٨.٣ التسوية المحاسبية

```
١. جرد فعلي للمخزن
   ↓
٢. مقارنة بالنظام
   ↓
٣. لو فيه فرق:
   - زيادة: حركة ADJ+ (قيد: مخزون ← إيراد تسوية)
   - نقصان: حركة ADJ- (قيد: مصروف تسوية ← مخزون)
   ↓
٤. التسوية بتعمل قيد محاسبي تلقائي
```

## ٩. القواعد والقيود

### ٩.١ قواعد عامة

| القاعدة | الشرح |
|---------|-------|
| ممنوع الرصيد السالب | ماينفعش تبيع اللي مش عندك |
| ممنوع القيد غير المتوازن | المدين لازم يساوي الداين |
| ممنوع تعديل قيد مرحل | لازم تعكسه وتعمل جديد |
| ممنوع تاريخ في المستقبل | كل الحركات لتاريخ اليوم أو قبله |
| ممنوع تجاوز حد الائتمان | النظام يوقف البيع الآجل |

### ٩.٢ قواعد المخزن

| الحالة | الإجراء |
|--------|---------|
| الرصيد = صفر | ينفع تبيع لو فيه حجز |
| الرصيد < المطلوب | ERROR: "الرصيد غير كافٍ" |
| التسوية بدون سبب | ERROR: "لازم ذكر سبب التسوية" |

### ٩.٣ قواعد الحسابات

| الحالة | الإجراء |
|--------|---------|
| القيد على حساب رئيسي | ERROR: "اختر حساب فرعي" |
| القيد غير متوازن | ERROR: "القيد غير متوازن" |
| تعديل قيد مرحل | ERROR: "القيد مرحل، استخدم عكس" |

## ١٣. ملخص سريع

| سؤال | الجواب |
|------|--------|
| النظام بيعمل إيه؟ | بيدير مخزنك، حساباتك، عملائك، مورديك، وتقاريرك |
| ليه FIFO؟ | عشان التكلفة تكون عادلة (اللي دخل أول يطلع أول) |
| ليه القيود المحاسبية؟ | عشان كل حركة ليها أثر مالي مسجل |
| ليه ممنوع الرصيد السالب؟ | عشان ماتبيعش حاجة مش عندك |
| ليه ممنوع تعديل القيد المرحل؟ | عشان سلامة البيانات المحاسبية |
| مين يستخدم النظام؟ | مدير، محاسب، بائع، مستلم مشتريات، أمين مخزن |
| أول حاجة تعملها؟ | إعدادات الشركة + شجرة الحسابات |
| آخر حاجة تعملها؟ | التقارير والطباعة |
